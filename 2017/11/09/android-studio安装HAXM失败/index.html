基本配置

先说一下基本配置

win10系统

android-studio用的是2.3.3版本，自带SDK的那个。安装完成之后，通过SDK Manager->SDK Tools安装了23.0.2这个老版本的android sdk，因为weex开发需要。

然后就开始踩坑了。。。

安装HAXM失败

就是打开一个项目（说得简单，中间又经历了各种辛酸，什么开weex项目要开platformx->android这个文件夹，什么building 'android' gradel跑一整天。。），然后配置AVD Manager（AVD的全称为：Android Virtual Device，就是Android运行的虚拟设备）之后，开始运行。。。

问题来了，提示运行这东西要安装HAXM，装嘛，可想而知，安装失败，直接说怎么个失败法吧。

通过Tools->Android->SDK Manager->SDK Tools选择下面这个安装：

装是装上了，但是运行安卓模拟器的时候还是提示没有安装HAXM，失败

解决方法

先进电脑的bios，把确认那个Intel Virtual Technology有没有开，没有的话就打开，下图这个是没有打开，设置成enable即可。

完成后，到intel的HAXM官网下载HAXM压缩包（这是它官方的安装教程）:

完成后，解压，运行里面的intelhaxm-android.exe文件安装HAXM，完成后。博主到这边就行了。再去android-studio里面运行安卓模拟器就成功了，没再提醒没安装HAXM。

怎么判断HAXM有没有安装成功？

打开cmd，运行sc query intelhaxm，出现这样的结果，就说明安装成功了：

注意

sc start intelhaxm用来启动HAXM，sc stop intelhaxm用来停止HAXM，用android-studio运行安卓模拟器之前，记得先启动HAXM
